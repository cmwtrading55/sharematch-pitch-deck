## All Issues

### High

**ISSUE-001: Mobile hero banner hidden behind nav**
- **Category:** visual / functional
- **Source:** browser (mobile 375x812)
- **Evidence:** Nav is 104px tall (logo h-16 = 64px + py-5 = 40px padding). The badge sits at 66px from top (hero `pt-16` = 64px + mb-6 margin). Overlap = 37px.
- **Fix difficulty:** easy
- **Root cause:** `StickyNav.tsx:68` - logo is `h-16` on all viewports. `HeroSection.tsx:19` - `pt-16` (64px) is insufficient clearance for the 104px nav on mobile.
- **Fix:** Either reduce mobile logo to `h-10` or `h-12` via responsive class (`h-10 md:h-16`), or increase hero top padding on mobile (`pt-28 md:pt-16`).

**ISSUE-002: Mobile sticky nav oversized on scroll**
- **Category:** visual / UX
- **Source:** browser (mobile 375x812, scrolled)
- **Evidence:** Scrolled nav is 73px tall with large opaque green background. On a 812px viewport, this consumes ~9% of the screen. The logo shrinks to `h-12` (48px) + `py-3` (24px) = 72px. On desktop this is fine but on mobile it's disproportionate.
- **Fix difficulty:** easy
- **Root cause:** `StickyNav.tsx:57-69` - no mobile-specific sizing for the scrolled state.
- **Fix:** Add mobile-specific classes: `h-8 md:h-12` when scrolled, and `py-2 md:py-3` when scrolled.

### Medium

**ISSUE-003: iPad (768px) nav "Invest Now" button clipped**
- **Category:** visual
- **Source:** browser (iPad 768x1024)
- **Evidence:** Screenshot shows the green "Invest Now" button partially cut off at the right edge. The `md:` breakpoint (768px) shows the desktop nav, but there isn't enough horizontal space for logo + 6 nav items + button.
- **Fix difficulty:** easy
- **Fix:** Either bump the nav breakpoint to `lg:` (1024px), or hide the "Invest Now" button at `md:` and only show at `lg:`.

**ISSUE-004: Heading order skip (H2 -> H4)**
- **Category:** accessibility
- **Source:** Lighthouse
- **Evidence:** Roadmap section has H2 "2026: The Inflection Year" followed by H4s ("Seed Fundraise", "KSA Platform Launch", etc.), skipping H3. Investment section similarly jumps H2 -> H4.
- **Fix difficulty:** trivial
- **Fix:** Change the roadmap milestone and investment H4 elements to H3.

**ISSUE-005: No linter configured**
- **Category:** code quality
- **Source:** static analysis
- **Evidence:** No `.eslintrc`, no `lint` script, build reports "Skipping linting".
- **Fix difficulty:** trivial
- **Fix:** Run `npx next lint --init` and add `"lint": "next lint"` to package.json scripts.
